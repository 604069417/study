🏠 React 基础概念

什么是React？

React是一个用于构建用户界面的JavaScript库，由Facebook开发并开源。它专注于视图层，让创建交互式UI变得轻而易举。React的核心思想是：将复杂的UI拆分成独立的、可复用的组件，每个组件管理自己的状态，当数据变化时，React会高效地更新和渲染组件。

🎯 通俗比喻

React就像乐高积木：想象你在搭建一座城堡，如果用一整块大石头雕刻，修改起来非常困难。但如果用乐高积木，你可以把城堡拆分成城墙、塔楼、门窗等独立部件，每个部件都可以单独修改、替换和复用。React组件就是这些"乐高积木"，你可以用它们搭建出复杂的网页应用。

React的四大核心特性

1. 声明式编程 重点

你只需要告诉React"想要什么"，而不是"怎么做"。React会自动处理DOM操作，当数据变化时，UI会自动更新。

📝 声明式 vs 命令式对比 
// ❌ 命令式编程（传统方式）：一步步告诉浏览器怎么做
const container = document.getElementById('container');
container.innerHTML = '';
const title = document.createElement('h1');
title.textContent = 'Hello';
container.appendChild(title);

// ✅ 声明式编程（React方式）：只描述UI应该是什么样子
function App() {
  return (
    <div>
      <h1>Hello</h1>
    </div>
  );
}
// React会自动处理DOM的创建和更新，你只需关注"是什么"，而非"怎么做"

2. 组件化 重点

将UI拆分成独立、可复用的组件，每个组件负责自己的逻辑和视图。组件可以嵌套组合，形成复杂的UI结构。

3. 虚拟DOM

React在内存中维护一个虚拟DOM树，当状态变化时，React会比较新旧虚拟DOM的差异，然后只更新需要变化的部分到真实DOM，这大大提高了性能。

🎯 虚拟DOM比喻

虚拟DOM就像装修图纸：假设你要重新装修房子，直接拆墙改结构太麻烦了。聪明的做法是先画两张图纸（旧布局和新布局），对比找出需要改动的地方，然后只改那些地方。虚拟DOM就是React的"图纸"，它先在内存中比较差异，再最小化操作真实DOM。

4. 单向数据流

数据从父组件流向子组件（通过props），子组件不能直接修改父组件的数据。这种单向流动让数据变化可预测，便于调试。

⚠️ 重点理解

为什么React要使用单向数据流？

  • 数据流向清晰，便于追踪数据变化来源
  • 避免双向绑定带来的数据混乱问题
  • 组件之间耦合度低，更容易维护和测试
  • 配合状态管理工具（如Redux）效果更佳

React应用的基本结构

📝 React应用入口文件 
// main.jsx 或 index.js - React应用的入口文件
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

// React 18 新的创建根节点的方式
// createRoot 用于创建一个React根容器
const root = ReactDOM.createRoot(
  document.getElementById('root')
);

// render方法将React组件渲染到DOM中
// StrictMode 是开发模式下的检查工具，帮助发现潜在问题
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

React 18引入了新的根API（createRoot），替代了旧的ReactDOM.render。新API启用了并发特性，为未来的React功能做好准备。StrictMode只在开发环境生效，不会影响生产环境。

🎨 JSX 语法详解

什么是JSX？

JSX是JavaScript的语法扩展，允许我们在JavaScript中编写类似HTML的代码。它让React组件的结构更加直观和易于理解。JSX不是模板语言，而是JavaScript的语法糖，最终会被编译成普通的JavaScript对象。

🎯 JSX比喻

JSX就像带超能力的HTML：普通的HTML只能写静态结构，而JSX像是给HTML装上了"大脑"，可以在里面写JavaScript逻辑、变量、表达式，让UI变得动态和智能。就像把HTML和JavaScript融合在一起，各取所长。

JSX的本质 重点

JSX在编译后会被转换成React.createElement()函数调用。以下两段代码完全等价：

📝 JSX编译前后对比 
// JSX写法（我们写的）
const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

// 编译后的JavaScript（React实际执行的）
const element = React.createElement(
  'h1',                    // 标签名
  {className: 'greeting'}, // 属性对象
  'Hello, world!'          // 子元素
);

// 最终生成的对象（虚拟DOM节点）
const element = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Hello, world!'
  }
};

JSX语法规则 重点

规则1：使用{}嵌入JavaScript表达式

📝 在JSX中使用JavaScript表达式 
function Welcome(props) {
  // 在JSX中，用花括号{}包裹JavaScript表达式
  const name = '张三';
  const age = 25;
  
  return (
    <div>
      {/* 变量 */}// 注意：注释也需要用花括号包裹
      <h1>你好，{name}</h1>
      
      {/* 表达式计算 */}
      <p>明年你{age + 1}岁</p>
      
      {/* 函数调用 */}
      <p>现在是：{new Date().toLocaleDateString()}</p>
      
      {/* 三元表达式 */}
      <p>{age >= 18 ? '成年人' : '未成年人'}</p>
      
      {/* 数组方法 - map会返回新数组，JSX会自动展开 */}
      <ul>
        {[1, 2, 3].map(n => <li key={n}>{n}</li>)}
      </ul>
    </div>
  );
}

规则2：class变成className，for变成htmlFor

📝 JSX中的属性命名 
// ❌ 错误写法：class和for是JavaScript保留字
<div class="container">
  <label for="name">姓名</label>
</div>

// ✅ 正确写法：使用className和htmlFor
<div className="container">
  <label htmlFor="name">姓名</label>
</div>

// 其他常见属性映射：
// tabindex → tabIndex
// readonly → readOnly
// colspan → colSpan
// onclick → onClick（事件用驼峰命名）

规则3：JSX必须有一个根元素

📝 JSX根元素规则 
// ❌ 错误：多个根元素
return (
  <h1>标题</h1>
  <p>内容</p>
);

// ✅ 方案1：用div包裹
return (
  <div>
    <h1>标题</h1>
    <p>内容</p>
  </div>
);

// ✅ 方案2：使用Fragment（不会产生额外DOM节点）
return (
  <>
    <h1>标题</h1>
    <p>内容</p>
  </>
);

// ✅ 方案3：使用React.Fragment（可以添加key属性）
return (
  <React.Fragment key="unique-key">
    <h1>标题</h1>
    <p>内容</p</React.Fragment>
);

规则4：标签必须正确闭合

📝 标签闭合规则 
// ❌ 错误：自闭合标签未关闭
<img src="photo.jpg">
<input type="text">
<br>

// ✅ 正确：自闭合标签必须用 / 关闭
<img src="photo.jpg" />
<input type="text" />
<br />
🔍 难点理解：JSX中的布尔值、null、undefined

在JSX中，true、false、null、undefined是合法的子元素，但它们不会被渲染。这个特性在条件渲染中非常有用：

// 以下都不会渲染任何内容
<div>{true}</div>
<div>{false}</div>
<div>{null}</div>
<div>{undefined}</div>

// 利用这个特性进行条件渲染
<div>
  {showHeader && <h1>标题</h1>}
  // 如果showHeader为false，整个表达式返回false，不会渲染任何内容
</div>

// 注意：数字0会被渲染！
<div>
  {items.length && <ItemList items={items} />}
  // 如果items为空数组，length为0，会渲染出"0"！
  // 正确写法：
  {items.length > 0 && <ItemList items={items} />}
</div>

🧩 组件基础

什么是组件？

组件是React的核心概念之一。组件将UI拆分成独立、可复用的部分，每个组件可以接收输入（props）并返回描述页面内容的React元素。组件让代码更加模块化、可维护和可测试。

🎯 组件比喻

组件就像函数：想象一个函数，它接收参数（props），处理数据，然后返回结果（JSX）。React组件本质上就是这样的函数，只不过返回的是UI描述而不是普通数据。就像你调用函数得到计算结果，你使用组件得到渲染的界面。

函数组件 vs 类组件 重点

React中有两种定义组件的方式：函数组件和类组件。现代React开发推荐使用函数组件配合Hooks。

📝 函数组件与类组件对比 
// ==================== 函数组件（推荐）====================
// 函数组件是一个纯函数，接收props作为参数，返回JSX
// 优点：代码简洁、易于测试、性能更好、可以使用Hooks
function Welcome(props) {
  // props是传入的属性对象，只读不可修改
  return <h1>你好，{props.name}</h1>;
}

// 箭头函数写法
const Welcome = (props) => (
  <h1>你好，{props.name}</h1>
);

// 解构props的写法（更常用）
function Welcome({ name, age }) {
  return (
    <div>
      <h1>你好，{name}</h1>
      <p>年龄：{age}</p>
    </div>
  );
}

// ==================== 类组件（旧写法）====================
// 类组件必须继承React.Component，必须有render方法
// 现在主要用于理解旧代码，新项目推荐使用函数组件
class Welcome extends React.Component {
  // render方法是必须的，返回JSX
  render() {
    // 通过this.props访问传入的属性
    return <h1>你好，{this.props.name}</h1>;
  }
}

组件的命名规范 重点

  • 组件名必须以大写字母开头：React会将小写字母开头的标签视为原生HTML标签
  • 使用有意义的名称：组件名应该描述其功能，如UserCard、NavigationBar
  • 一个文件一个组件：便于维护和复用
📝 组件命名规则 
// ❌ 错误：小写字母开头，React会认为这是HTML标签
function welcome(props) {
  return <h1>Hello</h1>;
}
<welcome />  // React会尝试调用 welcome()，而不是作为组件

// ✅ 正确：大写字母开头
function Welcome(props) {
  return <h1>Hello</h1>;
}
<Welcome />  // 正确：React知道这是一个组件

// ✅ 运行时确定组件类型（动态组件）
const components = {
  photo: PhotoStory,
  video: VideoStory
};

function Story({ storyType, story }) {
  // 动态选择组件，必须以大写字母开头的变量存储
  const SpecificStory = components[storyType];
  return <SpecificStory story={story} />;
}

组件的组合 重点

组件可以在其输出中引用其他组件，形成嵌套结构。这种组合方式是React的核心理念之一。

📝 组件组合示例 
// 子组件：Avatar（头像）
function Avatar({ src, alt, size = 'medium' }) {
  const sizeMap = {
    small: '32px',
    medium: '48px',
    large: '64px'
  };
  
  return (
    <img
      className="avatar"
      src={src}
      alt={alt}
      style={{ width: sizeMap[size], height: sizeMap[size] }}
    />
  );
}

// 子组件：UserInfo（用户信息）
function UserInfo({ user }) {
  return (
    <div className="user-info">
      // 在组件中使用其他组件
      <Avatar src={user.avatarUrl} alt={user.name} />
      <div className="user-name">{user.name}</div>
    </div>
  );
}

// 父组件：Comment（评论）
function Comment({ user, text, date }) {
  return (
    <div className="comment">
      // 组合多个子组件
      <UserInfo user={user} />
      <div className="comment-text">{text}</div>
      <div className="comment-date">
        {formatDate(date)}
      </div>
    </div>
  );
}

// 使用示例
const comment = {
  user: {
    name: '张三',
    avatarUrl: 'https://example.com/avatar.jpg'
  },
  text: '这是一条评论',
  date: new Date()
};

<Comment
  user={comment.user}
  text={comment.text}
  date={comment.date}
/>
⚠️ 重要原则：Props是只读的

React组件必须像纯函数一样，永远不要修改自己的props。这是React单向数据流的基础。如果需要修改数据，应该使用state。

// ❌ 错误：直接修改props
function BadComponent(props) {
  props.name = '新名字';  // 不要这样做！
  return <h1>{props.name}</h1>;
}

// ✅ 正确：使用state管理可变数据
function GoodComponent({ initialName }) {
  const [name, setName] = useState(initialName);
  return <h1>{name}</h1>;
}

📦 Props 属性传递

什么是Props？

Props（properties的缩写）是React组件的输入参数。父组件通过props向子组件传递数据，子组件通过props接收并使用这些数据。Props是只读的，子组件不能修改接收到的props。

🎯 Props比喻

Props就像函数参数：当你调用一个函数时，你传递参数给它。同样，当你使用一个组件时，你传递props给它。就像函数参数在函数内部是只读的一样，props在组件内部也是只读的——你不能改变传给你的东西，只能使用它。

Props的基本使用 重点

📝 Props传递与接收 
// ==================== 父组件 ====================
function App() {
  const user = { name: '张三', age: 25 };
  
  return (
    <div>
      // 传递字符串（使用引号）
      <Greeting name="李四" />
      
      // 传递JavaScript表达式（使用花括号）
      <Greeting name={user.name} />
      
      // 传递多个props
      <UserProfile
        name={user.name}
        age={user.age}
        isActive={true}
      />
      
      // 传递对象（使用展开运算符）
      <UserProfile {...user} />
      
      // 传递函数
      <Button onClick={() => console.log('点击了')} />
      
      // 传递组件作为props
      <Card header={<h2>标题</h2>}>
        内容
      </Card>
    </div>
  );
}

// ==================== 子组件 ====================
function Greeting(props) {
  // 方式1：通过props对象访问
  return <h1>你好，{props.name}</h1>;
}

function UserProfile({ name, age, isActive }) {
  // 方式2：解构props（推荐）
  return (
    <div>
      <h2>{name}</h2>
      <p>年龄：{age}</p>
      <p>状态：{isActive ? '活跃' : '离线'}</p>
    </div>
  );
}

默认Props 重点

当父组件没有传递某个prop时，可以使用默认值。

📝 设置默认Props 
// 方式1：解构时设置默认值（推荐）
function Button({ 
  text = '点击我',      // 字符串默认值
  type = 'button',    // 按钮类型默认值
  disabled = false,    // 布尔值默认值
  onClick = () => {}  // 函数默认值
}) {
  return (
    <button type={type} disabled={disabled} onClick={onClick}>
      {text}
    </button>
  );
}

// 使用示例
<Button />                          // 使用所有默认值
<Button text="提交" />              // 只覆盖text
<Button text="删除" disabled={true} /> // 覆盖多个

// 方式2：使用 defaultProps（旧写法，不推荐）
function Button(props) {
  return <button>{props.text}</button>;
}
Button.defaultProps = {
  text: '点击我'
};

children Props 重点

children是一个特殊的prop，用于传递组件的内容（子元素）。

📝 children的使用 
// 定义一个容器组件
function Card({ title, children }) {

      <div className="card-header">
        {title}
      </div>
      <div className="card-body">
        {children}
      </div>

// 使用示例
<Card title="用户信息">

// children可以是任何类型：元素、字符串、数字、函数等
<Card title="简单内容">

</Card>

<Card title="多个元素">
  <p>第一段</p>
  <p>第二段</p>

</Card>

💾 State 状态管理

什么是State？

State是组件内部的可变数据。与props不同，state由组件自己管理，当state变化时，组件会重新渲染。State是React响应式编程的核心，让组件能够"记住"信息并随时间变化。

🎯 State比喻

State就像组件的"记忆"：想象一个人（组件），props是他人的指令（别人告诉他的信息），而state是他自己的记忆。他可以回忆（读取state），也可以改变记忆（更新state）。当记忆改变时，他的行为也会改变（组件重新渲染）。

useState Hook 重点

useState是React提供的Hook，用于在函数组件中添加state。

📝 useState基础用法 
import { useState } from 'react';

function Counter() {
  // useState返回一个数组：[当前状态值, 更新状态的函数]
  // 参数0是state的初始值
  const [count, setCount] = useState(0);
  
  // 命名约定：使用[state, setState]的形式
  // count是当前值，setCount是更新函数
  
  return (
    <div>
      <p>当前计数：{count}</p>
      
      // 直接更新state
      <button onClick={() => setCount(count + 1)}>
        加1
      </button>
      
      // 函数式更新（基于前一个状态值）
      <button onClick={() => setCount(prev => prev + 1)}>
        加1（函数式）
      </button>
    </div>
  );
}

State更新的特点 重点 难点

📝 State更新机制详解 
function StateDemo() {
  const [count, setCount] = useState(0);
  
  // ❌ 错误：State更新是异步的，无法立即获取新值
  const handleClickWrong = () => {
    setCount(count + 1);
    console.log(count);  // 还是旧值！不是新值
  };
  
  // ✅ 正确：如果需要基于新值操作，使用useEffect
  useEffect(() => {
    console.log('count变化了：', count);
  }, [count]);
  
  // ❌ 错误：连续更新不会累加（因为闭包问题）
  const handleTripleWrong = () => {
    setCount(count + 1);  // 假设count是0
    setCount(count + 1);  // count还是0，结果是1
    setCount(count + 1);  // count还是0，结果是1
    // 最终count是1，不是3！
  };
  
  // ✅ 正确：使用函数式更新解决
  const handleTripleRight = () => {
    setCount(prev => prev + 1);  // 0 + 1 = 1
    setCount(prev => prev + 1);  // 1 + 1 = 2
    setCount(prev => prev + 1);  // 2 + 1 = 3
    // 最终count是3！
  };
  
  return <div>...</div>;
}
🔍 难点理解：State的不可变性

在React中，永远不要直接修改state对象，而是创建一个新的对象。这是React检测变化的基础。

function ObjectStateDemo() {
  const [user, setUser] = useState({ name: '张三', age: 25 });
  
  // ❌ 错误：直接修改state对象
  const updateAgeWrong = () => {
    user.age = 26;        // 直接修改，React检测不到变化！
    setUser(user);          // 引用没变，组件不会重新渲染
  };
  
  // ✅ 正确：创建新对象
  const updateAgeRight = () => {
    setUser({
      ...user,           // 复制原有属性
      age: 26            // 覆盖要修改的属性
    });
  };
  
  // ✅ 正确：使用函数式更新
  const updateAgeFunctional = () => {
    setUser(prev => ({
      ...prev,
      age: 26
    }));
  };
  
  // 数组也是同样的道理
  const [items, setItems] = useState([1, 2, 3]);
  
  // ❌ 错误：直接修改数组
  items.push(4);
  setItems(items);
  
  // ✅ 正确：创建新数组
  setItems([...items, 4]);
  // 或
  setItems(prev => [...prev, 4]);
}

🎯 事件处理

React事件处理的特点

React事件处理与原生DOM事件处理有一些区别。React使用合成事件（SyntheticEvent），它是对原生事件的跨浏览器封装，具有相同的接口。

⚠️ React事件与原生事件的区别
  • React事件使用驼峰命名：onClick、onSubmit、onFocus（而不是onclick、onsubmit）
  • 事件处理函数接收的是合成事件对象，不是原生事件
  • 不能通过return false阻止默认行为，必须使用e.preventDefault()

事件绑定方式 重点

📝 事件绑定的几种方式 
function EventDemo() {

📝 合成事件对象的使用 
function EventObjectDemo() {

    console.log(e.currentTarget); // 绑定事件的元素

  };
  
  // 注意：合成事件对象是池化的
  // 在异步代码中访问事件对象需要先保存
  const handleClickAsync = (e) => {

    setTimeout(() => {

🔀 条件渲染

什么是条件渲染？

在React中，你可以根据不同的条件渲染不同的UI。条件渲染与JavaScript中的条件判断一样，使用if语句、三元运算符或逻辑运算符。

条件渲染的几种方式 重点

📝 条件渲染的多种方式 
function ConditionalDemo({ isLoggedIn, user, messages }) {
  // ==================== 方式1：if语句 ====================
  // 适合逻辑复杂的情况
  let content;
  if (isLoggedIn) {
    content = <h1>欢迎回来！</h1>;
  } else {
    content = <h1>请先登录</h1>;
  }
  
  // 可以将条件判断提取为函数
  function renderContent() {
    if (!isLoggedIn) {
      return <LoginButton />;
    }
    if (user.isAdmin) {
      return <AdminPanel />;
    }
    return <UserDashboard />;
  }
  
  return (
    <div>
      {/* 方式1：使用变量 */}
      {content}
      
      {/* 方式2：调用函数 */}
      {renderContent()}
      
      {/* ==================== 方式2：三元运算符 ==================== */}
      {/* 适合简单的二选一情况 */}
      {isLoggedIn ? (
        <LogoutButton />
      ) : (
        <LoginButton />
      )}
      
      {/* 简洁写法 */}
      <span>{isLoggedIn ? '已登录' : '未登录'}</span>
      
      {/* ==================== 方式3：逻辑与(&&)运算符 ==================== */}
      {/* 适合"满足条件才渲染"的情况 */}
      {isLoggedIn && <UserMenu />}
      
      {/* 常见用法：显示未读消息数 */}
      {messages.length > 0 && (
        <span className="badge">
          {messages.length}条未读
        </span>
      )}
    </div>
  );
}

使用&&运算符时要注意，如果左侧条件是falsy值（如0），它会被渲染出来！

Key的作用与选择 重点 难点

🎯 Key的比喻

Key就像身份证号：想象一个班级的学生名单，每个学生有唯一的学号。当有新学生加入或学生离开时，老师通过学号来识别每个学生。同样，React通过key来识别列表中的每个元素，当列表发生变化时，React能高效地更新DOM。

⚠️ Key的重要性

Key帮助React识别哪些元素改变了、添加了或被删除了。正确的key选择对性能和正确性都很重要。

  • 使用稳定的唯一标识：如数据库ID、用户ID等
  • 避免使用索引作为key：当列表会重新排序、插入或删除时
  • key只需要在兄弟节点中唯一：不需要全局唯一
📝 Key的正确使用 
function KeyDemo() {
  const [items, setItems] = useState([
    { id: 'a1', text: '项目1' },
    { id: 'b2', text: '项目2' },
    { id: 'c3', text: '项目3' }
  ]);
  
  // ✅ 正确：使用唯一ID作为key
  const goodList = (
    <ul>
      {items.map(item => (
        <li key={item.id}>
          {item.text}
          <input type="text" />
        </li>
      ))}
    </ul>
  );
  
  // ❌ 错误：使用索引作为key（当列表会变化时）
  const badList = (
    <ul>
      {items.map((item, index) => (
        <li key={index}>  // 问题：插入/删除时key会错位
          {item.text}
        </li>
      ))}
    </ul>
  );
  
  // 什么时候可以用索引作为key？
  // 1. 列表是静态的，不会重新排序、过滤或添加
  // 2. 列表项没有id
  // 3. 列表不会用于计算或派生状态
  
  return goodList;
}
🔍 难点理解：为什么索引作为key会有问题？

当列表发生变化（如插入、删除、重新排序）时，使用索引作为key会导致React错误地匹配元素，可能产生以下问题：

  • 组件状态错乱（如输入框内容错位）
  • 性能下降（不必要的重新渲染）
  • 动画效果异常

🎯 受控组件比喻

🪝 Hooks 基础

什么是Hooks？

Hooks是React 16.8引入的新特性，它让你在不编写类组件的情况下使用state和其他React特性。Hooks是一些可以让你在函数组件里"钩入"React state及生命周期等特性的函数。

🎯 Hooks比喻

Hooks就像给函数组件装上"超能力"：普通的函数组件就像一个普通人，没有记忆（state），不知道什么时候出生或死亡（生命周期）。Hooks就像是给这个人装上了各种超能力装备：useState给他记忆，useEffect给他感知时间的能力，useContext给他心灵感应的能力...

Hooks的使用规则 重点

⚠️ 两条重要规则
  1. 只在最顶层使用Hook：不要在循环、条件或嵌套函数中调用Hook。确保总是在React函数的最顶层调用它们。
  2. 只在React函数中调用Hook：不要在普通的JavaScript函数中调用Hook。可以在React的函数组件中调用，也可以在自定义Hook中调用。

useState详解 重点

useState用于在函数组件中添加state。它返回一个数组：当前状态值和更新状态的函数。

📝 useState完整用法 
import { useState } from 'react';

function UseStateDemo() {
  // ==================== 基本用法 ====================
  const [count, setCount] = useState(0);
  
  // ==================== 惰性初始化 ====================
  // 初始值需要计算时，传入函数避免每次渲染都计算
  const [expensiveValue, setExpensiveValue] = useState(() => {
    // 这个函数只在初始渲染时执行一次
    return computeExpensiveValue();
  });
  
  // ==================== 更新方式 ====================
  // 方式1：直接传入新值
  const increment = () => setCount(count + 1);
  
  // 方式2：函数式更新（基于前一个状态）
  // 当新状态依赖旧状态时，推荐使用这种方式
  const incrementFunctional = () => setCount(prev => prev + 1);
  
  // ==================== 对象/数组状态 ====================
  const [user, setUser] = useState({ name: '', age: 0 });
  
  // 更新对象时，必须创建新对象（不可变性）
  const updateName = (newName) => {
    setUser(prev => ({ ...prev, name: newName }));
  };
  
  const [items, setItems] = useState([]);
  
  // 添加元素
  const addItem = (item) => {
    setItems(prev => [...prev, item]);
  };
  
  // 删除元素
  const removeItem = (id) => {
    setItems(prev => prev.filter(item => item.id !== id));
  };
  
  // 更新元素
  const updateItem = (id, newData) => {
    setItems(prev => prev.map(item => 
      item.id === id ? { ...item, ...newData } : item
    ));
  };
  
  return <div>...</div>;
}

useEffect详解 重点 难点

useEffect用于处理副作用，如数据获取、订阅、手动修改DOM等。它替代了类组件中的多个生命周期方法。

📝 useEffect完整用法 
import { useState, useEffect } from 'react';

function UseEffectDemo({ userId }) {
  const [data, setData] = useState(null);
  const [count, setCount] = useState(0);
  
  // ==================== 基本用法 ====================
  // 每次渲染后都执行
  useEffect(() => {
    console.log('组件渲染了');
  });
  
  // ==================== 空依赖数组 ====================
  // 只在挂载时执行一次（类似componentDidMount）
  useEffect(() => {
    console.log('组件挂载了');
  }, []); // 空数组表示不依赖任何值
  
  // ==================== 依赖特定值 ====================
  // 只在count变化时执行
  useEffect(() => {
    document.title = `点击了 ${count} 次`;
  }, [count]); // 依赖数组
  
  // ==================== 清理函数 ====================
  // 返回一个函数作为清理函数
  useEffect(() => {
    // 订阅
    const subscription = someAPI.subscribe((data) => {
      setData(data);
    });
    
    // 返回清理函数（类似componentWillUnmount）
    // 在组件卸载时或下次effect执行前调用
    return () => {
      subscription.unsubscribe();
    };
  }, [userId]); // userId变化时会先清理，再执行新effect
  
  // ==================== 数据获取 ====================
  useEffect(() => {
    let ignore = false;  // 用于处理竞态条件
    
    async function fetchData() {
      const result = await fetch(`/api/users/${userId}`);
      const json = await result.json();
      
      // 检查是否应该忽略这次结果
      if (!ignore) {
        setData(json);
      }
    }
    
    fetchData();
    
    return () => {
      ignore = true;  // 清理时设置标志
    };
  }, [userId]);
  
  return <div>...</div>;
}
🔍 难点理解：useEffect的依赖数组

依赖数组决定了useEffect何时执行。理解依赖数组是正确使用useEffect的关键。

  • 不传依赖数组：每次渲染后都执行
  • 空数组[]：只在挂载时执行一次
  • [dep1, dep2]：挂载时和dep1或dep2变化时执行

注意：如果依赖数组中使用了组件内的变量（props、state等），应该将它们全部包含在依赖数组中。可以使用ESLint的react-hooks插件来帮助检查。

useReducer 难点

useReducer是useState的替代方案，适用于复杂的状态逻辑。它使用Redux模式：通过dispatch派发action，reducer处理action并返回新状态。

📝 useReducer使用 
import { useReducer } from 'react';

// reducer函数：接收当前状态和action，返回新状态
function todosReducer(state, action) {
  switch (action.type) {
    case 'add':
      return [...state, { id: Date.now(), text: action.payload }];
    case 'delete':
      return state.filter(todo => todo.id !== action.payload);
    case 'toggle':
      return state.map(todo =>
        todo.id === action.payload
          ? { ...todo, completed: !todo.completed }
          : todo
      );
    default:
      throw new Error(`Unknown action: ${action.type}`);
  }
}

function TodoList() {
  // useReducer返回：[当前状态, dispatch函数]
  // 参数：reducer函数, 初始状态
  const [todos, dispatch] = useReducer(todosReducer, []);
  const [input, setInput] = useState('');
  
  const handleAdd = () => {
    // 派发action
    dispatch({ type: 'add', payload: input });
    setInput('');
  };
  
  return (
    <div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
      />
      <button onClick={handleAdd}>添加</button>
      
      <ul>
        {todos.map(todo => (
          <li key={todo.id}>
            {todo.text}
            <button onClick={() => dispatch({ type: 'delete', payload: todo.id })}>
              删除
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}

useRef 重点

useRef返回一个可变的ref对象，其.current属性被初始化为传入的参数。返回的ref对象在组件的整个生命周期内保持不变。

📝 useRef使用场景 
import { useRef, useState, useEffect } from 'react';

function UseRefDemo() {
  // ==================== 场景1：访问DOM元素 ====================
  const inputRef = useRef(null);
  
  const focusInput = () => {
    // 通过ref.current访问DOM元素
    inputRef.current.focus();
  };
  
  // ==================== 场景2：存储可变值（不触发重渲染）====================
  const [count, setCount] = useState(0);
  const renderCount = useRef(0);
  
  // 每次渲染时更新ref，但不会触发重渲染
  renderCount.current += 1;
  
  // ==================== 场景3：保存前一次的值 ====================
  const prevCountRef = useRef();
  
  useEffect(() => {
    // 在effect中保存当前值
    prevCountRef.current = count;
  }, [count]);
  
  const prevCount = prevCountRef.current;
  
  // ==================== 场景4：存储定时器ID ====================
  const intervalRef = useRef(null);
  
  useEffect(() => {
    intervalRef.current = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);
    
    return () => clearInterval(intervalRef.current);
  }, []);
  
  return (
    <div>
      <input ref={inputRef} />
      <button onClick={focusInput}>聚焦输入框</button>
      
      <p>当前值：{count}</p>
      <p>上一次的值：{prevCount}</p>
      <p>渲染次数：{renderCount.current}</p>
    </div>
  );
}

useMemo & useCallback 难点

这两个Hook用于性能优化，通过记忆化避免不必要的计算或函数重新创建。

📝 useMemo和useCallback对比 
import { useState, useMemo, useCallback } from 'react';

function MemoDemo({ items, onItemClick }) {
  const [count, setCount] = useState(0);
  
  // ==================== useMemo：记忆计算结果 ====================
  // 只有当items变化时才重新计算
  const sortedItems = useMemo(() => {
    console.log('重新排序');
    return [...items].sort((a, b) => a.name.localeCompare(b.name));
  }, [items]);  // 依赖数组
  
  // 复杂计算的记忆化
  const expensiveValue = useMemo(() => {
    // 假设这是一个耗时计算
    let result = 0;
    for (let i = 0; i < 1000000; i++) {
      result += i;
    }
    return result;
  }, []);  // 空依赖，只计算一次
  
  // ==================== useCallback：记忆函数 ====================
  // 返回一个记忆化的回调函数
  // 只有当依赖变化时，函数引用才会改变
  
  // 不使用useCallback：每次渲染都创建新函数
  const handleClickBad = () => {
    console.log(count);
  };
  
  // 使用useCallback：只有count变化时才创建新函数
  const handleClick = useCallback(() => {
    console.log(count);
  }, [count]);
  
  // 传递给子组件的回调函数
  // 如果不用useCallback，每次父组件渲染，子组件都会收到新的函数prop
  // 可能导致子组件不必要的重渲染
  const handleItemClick = useCallback((id) => {
    onItemClick(id);
  }, [onItemClick]);
  
  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>
        点击次数：{count}
      </button>
      <ItemList items={sortedItems} onItemClick={handleItemClick} />
    </div>
  );
}

自定义Hooks 重点 扩展

自定义Hook是一个函数，其名称以"use"开头，函数内部可以调用其他Hook。自定义Hook用于提取和复用组件逻辑。

📝 自定义Hook示例 
// ==================== 自定义Hook：useLocalStorage ====================
// 封装localStorage的状态管理逻辑
function useLocalStorage(key, initialValue) {
  // 惰性初始化：从localStorage读取初始值
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });
  
  // 包装setValue函数，同时更新localStorage
  const setValue = useCallback((value) => {
    try {
      // 支持函数式更新
      const valueToStore = value instanceof Function 
        ? value(storedValue) 
        : value;
      
      setStoredValue(valueToStore);
      localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  }, [key, storedValue]);
  
  return [storedValue, setValue];
}

// ==================== 自定义Hook：useFetch ====================
// 封装数据获取逻辑
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    let ignore = false;
    
    async function fetchData() {
      try {
        setLoading(true);
        const response = await fetch(url);
        const json = await response.json();
        
        if (!ignore) {
          setData(json);
          setError(null);
        }
      } catch (e) {
        if (!ignore) {
          setError(e);
        }
      } finally {
        if (!ignore) {
          setLoading(false);
        }
      }
    }
    
    fetchData();
    
    return () => { ignore = true; };
  }, [url]);
  
  return { data, loading, error };
}

// ==================== 使用自定义Hook ====================
function UserProfile({ userId }) {
  // 使用useLocalStorage
  const [theme, setTheme] = useLocalStorage('theme', 'light');
  
  // 使用useFetch
  const { data: user, loading, error } = useFetch(`/api/users/${userId}`);
  
  if (loading) return <div>加载中...</div>;
  if (error) return <div>出错了：</div>;
  
  return (
    <div className={theme}>
      <h1>{user.name}</h1>
      <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>
        切换主题
      </button>
    </div>
  );
}

🌐 Context API

什么是Context？

Context提供了一种在组件树中共享数据的方式，而不必显式地通过组件树的逐层传递props。Context主要用于共享那些对于组件树中许多组件来说是"全局"的数据，比如当前认证的用户、主题或首选语言。

🎯 Context比喻

Context就像"广播电台"：想象一个广播电台在播放节目，所有打开收音机的人都能听到。不需要有人一个一个传递消息，只要调到正确的频道就能接收。Context就像这个电台，Provider是发射器，useContext是收音机，任何组件只要"调频"就能获取数据。

Context的使用步骤 重点

📝 Context完整使用示例 
import { createContext, useContext, useState } from 'react';

// ==================== 步骤1：创建Context ====================
// createContext接受一个默认值参数
// 当组件不在任何Provider中时，会使用这个默认值
const ThemeContext = createContext({
  theme: 'light',
  toggleTheme: () => {}
});

// ==================== 步骤2：创建Provider组件 ====================
function ThemeProvider({ children }) {

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// ==================== 步骤3：在组件树上层使用Provider ====================
function App() {

}

🔗 Refs 引用

什么是Refs？

Refs提供了一种访问DOM节点或在React组件中创建的React元素的方式。在典型的React数据流中，props是父组件与子组件交互的唯一方式，但某些情况下需要直接操作DOM。

Refs的使用场景 重点

  • 管理焦点、文本选择或媒体播放
  • 触发强制动画
  • 集成第三方DOM库
  • 存储不触发重渲染的可变值

创建和使用Refs 重点

📝 Refs的多种创建方式 
import { useRef, createRef, useState, forwardRef, useImperativeHandle } from 'react';

// ==================== 方式1：useRef（函数组件推荐）====================
function TextInput() {
  // 创建ref
  const inputRef = useRef(null);
  
  const focusInput = () => {
    // 通过current访问DOM元素
    inputRef.current.focus();
  };

      // 将ref绑定到DOM元素

// ==================== 方式2：回调Refs ====================
function CallbackRefDemo() {
  let inputRef = null;
  
  // 回调函数会在组件挂载时调用，参数是DOM元素
  // 卸载时调用，参数是null
  const setInputRef = (element) => {
    inputRef = element;
    if (inputRef) {
      inputRef.focus();
    }
  };

}

// ==================== 方式3：forwardRef转发Ref ====================

const FancyInput = forwardRef((props, ref) => {

// ==================== useImperativeHandle ====================

const CustomInput = forwardRef((props, ref) => {
  const inputRef = useRef(null);

    focus: () => {
      inputRef.current.focus();

      inputRef.current.value = '';
    }

  • 避免过度使用Refs：能通过声明式方式实现的，就不要用Refs
  • 不要在渲染期间读取或写入ref.current，这会导致不可预测的行为
  • ref.current的更新不会触发重渲染

⚡ 性能优化

React性能优化的核心原则

React本身已经做了很多优化，但在大型应用中，我们还需要注意以下几点来确保应用性能。

React.memo 重点

React.memo是一个高阶组件，它对组件进行包装，使其在props不变时跳过渲染。

📝 React.memo使用 
import { memo, useState } from 'react';

// 普通组件：父组件更新时，即使props没变也会重新渲染
function SlowComponent({ data }) {
  console.log('SlowComponent渲染了');
  // 假设这里有一些耗时的渲染
  return <div>{data}</div>;
}

// 使用memo包装：只有props变化时才重新渲染
const MemoizedComponent = memo(function SlowComponent({ data }) {
  console.log('MemoizedComponent渲染了');
  return <div>{data}</div>;
});

// 自定义比较函数
const MemoizedWithCompare = memo(
  function MyComponent({ user }) {
    return <div>{user.name}</div>;
  },
  // 第二个参数：比较函数，返回true表示不需要重新渲染
  (prevProps, nextProps) => {
    return prevProps.user.id === nextProps.user.id;
  }
);

// 使用示例
function Parent() {
  const [count, setCount] = useState(0);
  const [data, setData] = useState('数据');
  
  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>计数：{count}</button>

      <Suspense fallback={<div>加载中...</div>}>
        <HeavyComponent />
      </Suspense>
      
      // 多个懒加载组件可以共享一个Suspense
      <Suspense fallback={<div>加载页面中...</div>}>
        <Dashboard />
      </Suspense>
    </div>
  );
}

// 配合路由使用
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

function AppWithRoutes() {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Suspense>
  );
}

高阶组件（HOC）难点 扩展

高阶组件是一个函数，它接收一个组件并返回一个新组件。HOC用于复用组件逻辑。

📝 高阶组件示例 
// 高阶组件：添加loading状态
function withLoading(WrappedComponent) {
  return function WithLoadingComponent({ isLoading, ...props }) {
    if (isLoading) {
      return <div>加载中...</div>;
    }
    // 传递所有props给被包装组件
    return <WrappedComponent {...props} />;
  };
}

// 使用HOC
const UserListWithLoading = withLoading(UserList);

// 高阶组件：添加认证检查
function withAuth(WrappedComponent) {
  return function WithAuthComponent(props) {
    const { user, isLoading } = useAuth();
    
    if (isLoading) {
      return <div>检查登录状态...</div>;
    }
    
    if (!user) {
      return <LoginPage />;
    }
    
    return <WrappedComponent {...props} user={user} />;
  };
}

// 组合多个HOC
const EnhancedComponent = withAuth(withLoading(MyComponent));

Render Props 扩展

Render Props是一种在React组件之间共享代码的技术，使用一个值为函数的prop来渲染内容。

📝 Render Props示例 
// Render Props组件：提供鼠标位置
function Mouse({ render }) {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  
  const handleMouseMove = (e) => {
    setPosition({ x: e.clientX, y: e.clientY });
  };
  
  return (
    <div onMouseMove={handleMouseMove}>
      {/* 调用render函数，传入状态 */}render(position)}
    </div>
  );
}

// 使用Render Props
function App() {
  return (
    <Mouse
      render={({ x, y }) => (
        <h1>鼠标位置：({x}, {y})</h1>
      )}
    />
  );
}

// 使用children作为render prop（更常用）
function Mouse({ children }) {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  
  return (
    <div onMouseMove={handleMouseMove}>
      {children(position)}
    </div>
  );
}

// 使用方式
<Mouse>
  {({ x, y }) => <h1>鼠标位置：({x}, {y})</h1>}
</Mouse>
